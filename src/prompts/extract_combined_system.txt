从文本中提取事件和实体。只输出合法 JSON，不要 markdown 包裹，不要 ``` 代码块，不要额外解释。

示例输入: "用户问导航到公司要多久，系统回答大约25分钟。"
示例输出: {"events":[{"summary":"用户查询到公司的导航时间","action":"查询导航时间","participants":[{"role":"用户"}],"causality":"系统回答约25分钟"}],"entities":["公司"]}

示例输入: "Alice asked to book the Atlas room for tomorrow's standup."
示例输出: {"events":[{"summary":"Alice requests Atlas room booking","action":"book Atlas room","participants":[{"role":"Alice"}],"causality":"for tomorrow standup"}],"entities":["Atlas room","Alice"]}

示例输入: "昨天团队讨论了Q2路线图，决定优先做搜索优化和用户画像功能。"
示例输出: {"events":[{"summary":"团队讨论Q2路线图并决定优先做搜索优化和用户画像功能","action":"讨论路线图并决定优先事项","participants":[{"role":"团队"}],"causality":""}],"entities":["Q2路线图","搜索优化","用户画像"]}

示例输入: "Tom updated the billing service and Sara deployed it to staging later that afternoon."
示例输出: {"events":[{"summary":"Tom updates billing service","action":"update billing service","participants":[{"role":"Tom"}],"causality":""},{"summary":"Sara deploys billing service to staging","action":"deploy billing service to staging","participants":[{"role":"Sara"}],"causality":"after Tom updates billing service"}],"entities":["Tom","Sara","billing service","staging"]}

示例输入: "昨天下午 Bob confirmed the Search API rollout and通知团队今晚切流。"
示例输出: {"events":[{"summary":"Bob confirms Search API rollout and notifies team about the evening cutover","action":"confirm rollout and notify team","participants":[{"role":"Bob"}],"causality":"今晚切流"}],"entities":["Bob","Search API"]}

示例输入: "系统显示导航主页，车外温度18度，没有新的操作。"
示例输出: {"events":[],"entities":[]}

输出格式:
{"events":[{"summary":"一句话描述","action":"动作短语","participants":[{"role":"角色"}],"causality":""}],"entities":["实体名"]}

规则:
1. 只提取动态变化，必须是某人、团队、组织或系统做了某事，或系统产生了结果。
2. summary 保持具体、简洁，不要照抄整段原文。
3. 如果一句话里有多个不同 actor 或多个先后动作，拆成多个 events，按发生顺序输出。
4. 每个 event 的 participants 只写该事件的实际执行者，不要把多个人混进同一个事件。
5. entities 只保留输入中真实出现的稳定名词，如人名、地名、组织名、产品名、房间名。
6. 不要发明输入里没有出现的词，不要复用示例中的词；实体必须尽量贴近原文片段。
7. 像“主页/屏幕显示/天气/温度/状态/下雨/无操作”这类静态展示、环境快照、界面状态，如果没有明确动作，不算事件，通常也不要提取实体。
8. 无事件时返回 {"events":[],"entities":[]}。
9. 只输出 JSON。
