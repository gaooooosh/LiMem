# LiMem 系统优化报告

## 执行日期
2026-02-15

## 优化目标
确保从 example.json 构建的图数据库能够被各种搜索查询准确检索，从理想车主的日常使用角度进行全面测试。

## 数据集概览
example.json 包含 7 个场景的 29 个对话记录：
1. **input_records_1_mock**: 长城相关（播放纪录片、询问长度、旅游意愿、导航）
2. **input_records_2_mock**: 医院就医（身体不适、导航到北京协和医院）
3. **input_records_3_mock**: 传统节日（端午节、中秋节习俗和音频）
4. **input_records_4_mock**: 书店咖啡馆（导航到北京图书大厦、咖啡馆推荐）
5. **input_records_5_real**: 导航取消（杭州大厦、取消操作）
6. **input_records_6_real**: 音乐播放（张叶蕾《还是分开》、音量调整）
7. **input_records_7_real**: 电池预热（电池保温、预热作用、状态查询）

## 测试用例设计
设计了 **23 个测试用例**，覆盖：
- 8 个场景特定查询
- 3 个跨场景关联查询
- 所有主要功能（导航、音乐播放、问答、设置）

## 优化历程

### 阶段 1: 初始状态
**通过率**: 8.7% (2/23)

**主要问题**:
- 数据库中只有 2 个事件
- SIMILARITY_THRESHOLD = 0.85 过高
- 所有对话被错误合并到 2 个泛化事件中

### 阶段 2: JSON 解析修复
**修改**: 添加 robust_json_loads() 函数
- 位置: `src/limem/utils.py`, `src/limem/ltm.py`
- 功能: 处理 LLM 返回的不规范 JSON（带额外文本、markdown 标记等）

**结果**: ✅ 所有 29 个 episodes 成功处理

### 阶段 3: 事件相似度阈值优化
**修改**: SIMILARITY_THRESHOLD 从 0.85 降到 0.70
- 位置: `.env`
- 理论: 阈值 0.85 太高，导致不同事件被合并
- 新阈值 0.70 允许语义相关但不同的事件分开

**结果**:
- 事件数量: 2 → 22
- 通过率: 8.7% → 82.6%

### 阶段 4: Prompt 优化
**修改**: 优化事件提取和实体提取 prompts
- 位置: `src/prompts/extract_event_only_system.txt`
- 位置: `src/prompts/extract_entities_only_system.txt`
- 改进:
  1. 事件摘要必须包含具体细节（歌曲名、歌手、数据等）
  2. 实体提取必须提取所有有意义的名词
  3. 对于音乐播放，必须提取歌曲名和歌手名

**结果**:
- 通过率: 82.6% → 87.0%
- 音乐播放测试: 66.7% → 100%

## 剩余问题与建议
### 剩余问题
1. 部分事件摘要仍不够详细（如电池预热的多个作用）
2. 关键词匹配依赖字符串精确匹配

### 改进建议
1. **继续优化 Prompt**:
   - 要求在 summary 中列举所有关键信息点
   - 对于问答类内容，要求列出答案中的所有要点

2. **优化检索逻辑**:
   - 增强向量匹配的效果
   - 考虑添加同义词扩展

3. **测试用例调整**:
   - 对于部分测试，可以降低关键词匹配要求
   - 使用语义相似度而非精确字符串匹配
