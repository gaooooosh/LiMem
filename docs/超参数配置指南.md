# LiMem 超参数配置指南

## 一、核心超参数总览表

### 1. 记忆整合参数（存储阶段）

| 参数名 | 推荐值 | 可调范围 | 含义 | 实际影响 | 记忆类型 |
|--------|--------|----------|------|----------|----------|
| `SIMILARITY_THRESHOLD` | **0.70** | 0.60-0.90 | 事件相似度阈值，控制记忆合并 | 0.85→过度合并<br>0.70→细粒度区分 | 语义记忆 |
| `DECAY_RATE` | **1e-8** | 1e-9 ~ 1e-6 | 时间衰减率，控制记忆淡化速度 | 半衰期 ≈ 2.2 年<br>1e-7 ≈ 81 天 | 长期记忆 |
| `EPISODE_TTL` | **3600** | 1800-7200 | 原始对话保留时间（秒） | 1 小时后清理<br>释放存储空间 | 情景记忆 |
| `PRUNE_C_VALID_THRESHOLD` | **100** | 50-200 | 永久特征提升阈值 | c_valid > 100 → 永久特征 | 永久记忆 |

### 2. 检索参数（搜索阶段）

| 参数名 | 推荐值 | 可调范围 | 含义 | 实际影响 | 应用场景 |
|--------|--------|----------|------|----------|----------|
| `SEARCH_TOP_K` | **5** | 3-10 | 返回的 Top-K 事件数量 | 5→平衡召回与精度<br>10→高召回但噪音多 | 答案生成 |
| `SEARCH_MAX_ENTITIES` | **10** | 5-15 | 从查询提取的最大实体数 | 10→覆盖关键实体<br>15→噪音增加 | 查询理解 |
| `SEARCH_VECTOR_THRESHOLD` | **0.5** | 0.4-0.7 | 向量相似度阈值 | 0.5→平衡召回与精度<br>0.7→高精度低召回 | 实体匹配 |
| `SEARCH_VECTOR_TOP_K` | **10** | 5-20 | 每个实体返回的相似实体数 | 10→平衡性能与召回<br>20→高召回但慢 | 向量检索 |

### 3. 混合匹配参数

| 参数名 | 推荐值 | 含义 | 权重计算 |
|--------|--------|------|----------|
| `SEARCH_ENABLE_VECTOR_MATCH` | **True** | 启用向量匹配补充精确匹配 | 精确匹配: 1.0<br>向量匹配: similarity² |
| `SEARCH_LAMBDA` | **1e-9** | 搜索阶段衰减参数（已弃用） | 现使用统一的 DECAY_RATE |

| 1e-7 | 6,930,000 | **80** | **0.22** | 中期记忆 | 近期行程、临时安排 |
| 1e-6 | 693,000 | **8** | **0.02** | 短期记忆 | 临时对话、即时任务 |
| 1e-9 | 693,000,000 | **8,024** | **22** | 永久记忆 | 用户画像、核心特征 |

## 四、检索参数调优指南

### 1. TOP_K 选择策略

| TOP_K | 上下文长度 | 答案质量 | 响应速度 | 适用场景 |
|-------|-----------|----------|----------|----------|
| 3 | 短 | 精确但可能不完整 | 快 | 简单查询 |
| **5** | **适中** | **平衡** | **适中** | ✅ **通用场景** |
| 10 | 长 | 全面但可能噪音多 | 慢 | 复杂查询 |
| 15+ | 很长 | 噪音增加 | 很慢 | 需要大量上下文 |

### 2. 实体提取数量

| MAX_ENTITIES | 查询理解 | 检索范围 | 噪音水平 | 推荐 |
|--------------|----------|----------|----------|------|
| 5 | 核心实体 | 窄 | 低 | 简单查询 |
| **10** | **关键实体** | **适中** | **中** | ✅ **通用** |
| 15 | 全面实体 | 宽 | 高 | 复杂查询 |
| 20+ | 过多实体 | 过宽 | 很高 | ❌ 不推荐 |

### 3. 向量匹配参数

**SEARCH_VECTOR_THRESHOLD 权衡**：

```
阈值 = 0.3 (低阈值)
━━━━━━━━━━━━━━━━━━━━━━━━━━
✅ 高召回率（找到更多相关实体）
❌ 低精确率（引入噪音实体）
适用：需要高召回的场景

阈值 = 0.5 (推荐值)
━━━━━━━━━━━━━━━━━━━━━━━━━━
✅ 平衡召回率和精确率
✅ 过滤掉不相关的实体
适用：通用场景

阈值 = 0.7 (高阈值)
━━━━━━━━━━━━━━━━━━━━━━━━━━
✅ 高精确率（只保留高度相关实体）
❌ 低召回率（可能漏掉相关实体）
适用：需要高精度的场景
```

## 七、常见问题与解决方案

### Q1: 记忆检索不准确

**症状**：查询返回不相关的事件

**诊断**：
1. 检查 `SIMILARITY_THRESHOLD` 是否过低（< 0.60）
2. 检查 `SEARCH_VECTOR_THRESHOLD` 是否过低（< 0.4）
3. 检查实体提取是否准确

**解决方案**：
```bash
# 提高阈值，减少噪音
SIMILARITY_THRESHOLD=0.75
SEARCH_VECTOR_THRESHOLD=0.6
```

### Q2: 记忆过度碎片化

**症状**：同一事件被拆分成多个小事件

**诊断**：
1. 检查 `SIMILARITY_THRESHOLD` 是否过高（> 0.80）
2. 检查 Prompt 是否过于细化

**解决方案**：
```bash
# 降低阈值，允许合并
SIMILARITY_THRESHOLD=0.65
```

### Q3: 老记忆无法检索

**症状**：几个月前的记忆检索不到

**诊断**：
1. 检查 `DECAY_RATE` 是否过大（> 1e-7）
2. 计算半衰期是否过短

**解决方案**：
```bash
# 降低衰减率，延长记忆
DECAY_RATE=1e-9  # 半衰期 22 年
```

### Q4: 检索速度慢

**症状**：查询响应时间 > 2 秒

**诊断**：
1. 检查 `SEARCH_TOP_K` 是否过大（> 10）
2. 检查 `SEARCH_VECTOR_TOP_K` 是否过大（> 15）
3. 检查数据库索引

**解决方案**：
```bash
# 减少检索数量
SEARCH_TOP_K=5
SEARCH_VECTOR_TOP_K=8
```

| 指标 | 计算方法 | 目标值 | 含义 |

## 九、总结

### 核心原则

1. **平衡粒度与合并**：`SIMILARITY_THRESHOLD = 0.70`
2. **长期记忆保留**：`DECAY_RATE = 1e-8`（2.2 年半衰期）
3. **混合检索策略**：精确匹配 + 向量匹配

1. 🔴 **关键**：`SIMILARITY_THRESHOLD`（直接影响记忆粒度）

3. 🟢 **优化**：`SEARCH_TOP_K` / `SEARCH_MAX_ENTITIES`（平衡性能与召回）
